# Agent流程遵循问题分析

## 问题诊断

### 当前问题：Agent没有严格遵循三省六部制工作流程

#### 从thinking日志看，出现了以下违规：

```
用户请求：帮我创建一个系统包含页面和实际的验证方法

❌ 问题 1: 皇帝(huangdi)跳过了"声明任务"步骤
  应该做：我现在声明开始 "understand" 任务
  实际做：直接进入思维分析

❌ 问题 2: 中书省(zhongshu)跳过了"制定详细计划"步骤
  应该做：分析需求 → 研究现状 → 设计详细计划（5-10步）→ 风险分析
  实际做：在 thinking 中零散地进行分析，没有正式提交计划

❌ 问题 3: 门下省(menxia)没有审核任何东西
  应该做：[OK] 批准 / [CYCLE] 需要修改 / [FAIL] 拒绝
  实际做：完全跳过审核环节

❌ 问题 4: 任务队列被忽视
  应该做：@getTaskQueue 查看任务 → claimTask 声明任务
  实际做：没有与任务队列系统交互

❌ 问题 5: 过度思考(overthinking)
  思维链中：大量推测、假设、规划
  应该做：通过正式流程，一步步执行
```

## 工作流程对比

### ❌ 错误的做法（当前问题）

```
用户输入
  ↓
Claude 的思维链：
  ├─ 自己分析需求 (thinking日志)
  ├─ 自己规划步骤 (Todo列表)
  ├─ 自己假设项目结构
  ├─ 自己假设可能的实现
  └─ 最后... 什么都没执行

结果：❌ 无法执行，Agent流程被绕过
```

### ✅ 正确的做法（三省六部制）

```
用户输入
  ↓
皇帝(huangdi)：
  ├─ 理解需求
  ├─ 【声明】我现在开始 understand 任务
  └─ 提交理解报告

中书省(zhongshu)：
  ├─ 【声明】我现在开始 plan 任务
  ├─ 分析需求（深入）
  ├─ 研究现状（glob/grep/read）
  ├─ 设计详细计划（5-10步）
  ├─ 分析风险（3个+）
  └─ 提交计划

门下省(menxia)：
  ├─ 【声明】我现在开始 menxia_review 任务
  ├─ 深入理解计划
  ├─ 分析至少5个风险
  └─ 【决策】[OK] 批准 / [CYCLE] 修改 / [FAIL] 拒绝

尚书省(shangshu)：
  ├─ 【声明】我现在开始 execute 任务
  ├─ 严格遵循计划
  ├─ 逐步执行和验证
  └─ 完成并报告

皇帝(huangdi)：
  ├─ 【声明】我现在开始 verify 任务
  ├─ 验证完成质量
  └─ 【完成】任务完毕

结果：✅ 流程完整，责任清晰，质量有保证
```

## 问题根源

### 1. 思维链（Thinking）vs 正式流程

**思维链的问题：**
- 只在 Claude 脑子里思考
- 用户看不到分析过程
- 无法中途纠正
- 容易推翻前面的假设
- 最后可能什么都没执行

**正式流程的优势：**
- 用户可以看到每个步骤
- 可以在中途提出意见
- 有明确的审核和批准
- 有责任划分和追踪
- 最后一定会执行

## 应该做的改进

[执行步骤]

2. 创建新页面组件
3. 添加验证方法
4. 集成到路由
5. 编写演示代码
6. 测试验证功能

[预期成果]
- 一个功能完整的系统包含页面

[潜在风险]

## 核心规则（必须遵守）

### ✅ 每次AI开始工作前

```
1. 【声明任务】
   "我现在声明开始 [任务名] 任务"

2. 【按流程操作】
   understand → plan → menxia_review → execute → verify

3. 【与任务队列交互】
   @getTaskQueue 查看
   说"我声明 XXX 任务" 来 claim
   说"XXX 完成" 来标记完成

4. 【不跳过关键步骤】
   ❌ 不能跳过 menxia_review
   ❌ 不能跳过 plan
   ❌ 不能跳过 understand
```

### ❌ 禁止行为

```
❌ 在 thinking 中进行大量规划
  应该：通过正式的 plan 任务

❌ 假设和推测项目结构
  应该：用 read/glob/grep 真正查证

❌ 跳过审核直接执行
  应该：必须等 menxia 批准

❌ 自作主张修改计划
  应该：等用户反馈后修改
```

首先，我需要研究当前项目...

[详细计划]

## 总结

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| 流程被绕过 | thinking中进行规划 | 在正式流程中进行规划 |
| 无法执行 | 跳过了中间步骤 | 必须经过understand→plan→review→execute→verify |
| 用户无法跟进 | 只在Claude脑子里思考 | 将每一步都显示给用户 |
| 质量无保证 | 没有menxia审核 | 必须等审核通过 |
| 假设太多 | 没有实际查证 | 用read/glob/grep验证 |
