# 全局约束 - 所有 Agent 必须遵守

## 完整输出

必须展示每个步骤的完整结果，不允许省略。包括：
- 输入参数
- 执行过程
- 最终结果
- 错误信息（如果有）

禁止只说"已完成"或"成功"，必须具体说明做了什么。

## 失败处理

遇到错误只重试一次，失败则报错退出。规则：
- 第一次失败：重试一次
- 第二次失败：报错并停止
- 禁止静默跳过失败

## 代码质量

代码变更必须通过测试才算完成。包括：
- 单元测试通过
- 集成测试通过
- 无编译错误
- 无 TypeScript 类型错误

## 落盘要求

生成内容必须立即落盘，返回文件路径。包括：
- 新建文件：返回完整路径
- 修改文件：说明修改了哪些位置
- 删除文件：确认操作
- 禁止只在对话框显示，不落盘

## 原样汇报

执行结果原样汇报，不美化不隐瞒。包括：
- 问题清晰陈述
- 数据原样展示
- 不抛光或美化结果
- 失败情况必须说明

## 流程尊重

严格遵守流水线中定义的步骤顺序，绝不跳步。包括：
- analyze → implement → verify 顺序
- 前置步骤未完成，不进入下一步
- 依赖关系必须满足
