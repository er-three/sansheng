# Ionic 8 + Angular Standalone Project Standards
> Generated by OpenCode Agent for Reverse Engineering TDD Process

## 1. Project Architecture
- **Framework**: Ionic 8 + Angular (Latest)
- **Component Style**: **Standalone Components** (`standalone: true`)
- **Module System**: No `NgModule` for features. Use `imports: []` in `@Component`.
- **Change Detection**: `ChangeDetectionStrategy.OnPush` (Recommended for performance)
- **Dependency Injection**: Use `inject()` function instead of constructor injection where possible.

## 2. Directory Structure
```
src/
  app/
    pages/                  # Routable pages (Views)
      [feature-name]/       # Feature directory (kebab-case)
        [feature].page.ts
        [feature].page.html
        [feature].page.scss
        [feature].page.spec.ts
    components/             # Shared UI components
      [component-name]/
        [component].component.ts
        [component].component.html
        [component].component.scss
        [component].component.spec.ts
    services/               # Services + Providers
      [service-name].service.ts
      [service-name].service.spec.ts
    models/                 # Interfaces & Types
      [model-name].model.ts
    mocks/                  # Test mock data
      mock-[entity-name].ts
    pipes/
      [pipe-name].pipe.ts
    directives/
      [directive-name].directive.ts
    guards/
      [guard-name].guard.ts
```

## 3. Naming Conventions
| Type | File Pattern | Class Name | Selector |
|------|--------------|------------|----------|
| **Page** | `*.page.ts` | `*Page` | `app-*` |
| **Component** | `*.component.ts` | `*Component` | `app-*` |
| **Service** | `*.service.ts` | `*Service` | N/A |
| **Guard** | `*.guard.ts` | `*Guard` | N/A |
| **Interface** | `*.model.ts` | `*` (PascalCase) | N/A |
| **Test** | `*.spec.ts` | N/A | N/A |

### Specific Rules
- **Files**: `kebab-case` (e.g., `user-profile.page.ts`)
- **Classes**: `PascalCase` (e.g., `UserProfilePage`)
- **Variables/Methods**: `camelCase` (e.g., `getUserData()`)
- **Constants**: `UPPER_SNAKE_CASE` (e.g., `MAX_RETRY_COUNT`)

## 2A. Path Resolution Table（路径解析查找表）

> ⚠️ **创建任何文件前必须查此表，不允许自行推断路径。**

### 转换算法（Name → File Path）

**Step 1：识别类型**，从 `infrastructure.yaml` 的 `type` 字段判断：

| 类型关键词 | 归类 | 目标目录 | 文件后缀 | CLI 命令 |
|-----------|------|---------|---------|---------|
| 包含 `Service`（非 Provider） | Service | `src/app/services/` | `.service.ts` | `ionic g service` |
| 包含 `Provider` | Service | `src/app/services/` | `.service.ts` | `ionic g service` |
| 包含 `Component` | Component | `src/app/components/[name]/` | `.component.ts` | `ionic g component` |
| 包含 `Pipe` | Pipe | `src/app/pipes/` | `.pipe.ts` | `ionic g pipe` |
| 包含 `Directive` | Directive | `src/app/directives/` | `.directive.ts` | `ionic g directive` |
| 包含 `Guard` | Guard | `src/app/guards/` | `.guard.ts` | `ionic g guard` |
| Interface / Model | Model | `src/app/models/` | `.model.ts` | 直接 write |
| Mock 数据 | Mock | `src/app/mocks/` | `.ts` | 直接 write |

**Step 2：类名 → 文件名转换规则**（camelCase / PascalCase → kebab-case）

```
转换规则：
1. 去掉类型后缀（Service / Provider / Component / Pipe / Directive / Guard）
2. 将剩余部分从 PascalCase/camelCase 转为 kebab-case
3. 拼接目录 + 文件名 + 类型后缀

示例：
  soapTaskProxy     → soap-task-proxy     → src/app/services/soap-task-proxy.service.ts
  databaseService   → database            → src/app/services/database.service.ts
  StorageProvider   → storage             → src/app/services/storage.service.ts
  LoadingProvider   → loading             → src/app/services/loading.service.ts
  CommonService     → common              → src/app/services/common.service.ts
  getLanguageCode   → get-language-code   → src/app/services/get-language-code.service.ts
  CommonHeader      → common-header       → src/app/components/common-header/common-header.component.ts
  LoginInformation  → login-information   → src/app/components/login-information/login-information.component.ts
  MaintenanceItem   → maintenance-item    → src/app/models/maintenance-item.model.ts
  BuildingItem      → building-item       → src/app/models/building-item.model.ts
  PlanUnit          → plan-unit           → src/app/models/plan-unit.model.ts
```

**Step 3：类名规则**（文件名 → 类名）

```
文件名反向推导类名规则：
  kebab-case 每段首字母大写 + 类型后缀（Service / Component 等）

示例：
  soap-task-proxy.service.ts   → class SoapTaskProxyService
  database.service.ts          → class DatabaseService
  storage.service.ts           → class StorageService
  common-header.component.ts   → class CommonHeaderComponent
  maintenance-item.model.ts    → interface MaintenanceItem
```

## 4. Coding Standards (Ionic 8)
- **Imports**: Use `@ionic/angular/standalone` for Ionic components.
  ```typescript
  import { IonHeader, IonContent, ... } from '@ionic/angular/standalone';
  
  @Component({
    standalone: true,
    imports: [IonHeader, IonContent, ...]
  })
  ```
- **Forms**: Use `ReactiveFormsModule` over `FormsModule`.
- **Routing**: Use `loadComponent` for lazy loading routes.
  ```typescript
  {
    path: 'home',
    loadComponent: () => import('./pages/home/home.page').then(m => m.HomePage)
  }
  ```

## 5. TDD Workflow (Reverse Engineering)
1.  **Red (Test)**: Create `*.spec.ts` based on `behavior.md`. Define `describe` blocks for scenarios and `it` blocks for steps. Assertions should fail initially.
2.  **Green (Code)**: Implement minimal logic in `*.ts` and template in `*.html` to pass tests. Use `ui.yaml` for structure and `logic.ts` for logic.
3.  **Refactor**: Optimize code, add types, extract methods, apply `design-spec.md` styles.

## 6. Testing Strategy
- **Framework**: Jasmine + Karma
- **Unit Tests**: Focus on component logic (`logic.ts`) and DOM interaction (`ui.yaml` bindings).
- **Mocks**: All external services (API, DB, Storage) must be mocked using `jasmine.createSpyObj`.

## 7. Migration/Reverse Engineering Specifics
- **Legacy Logic**: `logic.ts` contains extracted legacy logic. It must be adapted to **Standalone** style (e.g., remove `NavParams`, use `@Input` or Router state).
- **Legacy UI**: `ui.yaml` contains abstract UI. Map to Ionic 8 components (e.g., `<ion-button>` instead of `<button ion-button>`).
